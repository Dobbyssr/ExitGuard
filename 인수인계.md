## 1. ExitGuard가 무엇인가

### 한 줄 정의
> 직원의 퇴사·해고가 발생하는 순간, AI가 하나의 이벤트를 **세 레일로 동시 관제**한다 —
> **① 노무**(절차·법정기한·분쟁 리스크 알림) **② 영업비밀**(그 사람이 접근한 핵심 자산의 보호요건 점검·재서약) **③ 보안**(계정·SaaS·자산 회수) — 그리고 전 과정을 증적으로 봉인한다.

### 제품 성격 (2줄로 기억할 것)
1. **판단·대리·서류작성을 하지 않는다.** 공개된 법령·판례 기준과 회사 내부 상태의 **"대조 결과·기한·기록"만** 제공한다.
2. **기존 시스템을 바꾸지 않는다.** read-only 관제 계층으로 올라탄다(그룹웨어·DLP 로그 read-only 흡수).

### 왜 통합인가 (1+1=3의 논리 — 심사 방어의 핵심)
- **퇴사는 세 리스크의 교차점.** 노무분쟁도, 기술유출도, 보안사고도 터지는 날은 같은 날 = 퇴사일.
- **영업비밀 소송의 승부 증적은 퇴사 시점에 생성된다.** 법원이 보는 것(퇴직 시 비밀유지 재서약, 접근 자산 특정, 권한 회수 시점, 자료 반납 확인)이 전부 퇴사 처리 과정의 기록이다.
- 이 통합 논리가 **"이것저것 붙인 잡탕 아니냐"** 공격에 대한 유일한 방어다. "기능을 모은 게 아니라 **이벤트를 중심에 뒀다**."

## 2. 왜 필요한가 (검증된 근거 — 피치·카피에 그대로 사용 가능)

> 아래 수치는 실제 언론·판례·공식통계 기반. 상세·출처·검증노트는 원본 리서치 참조(§8).

| 레일 | 핵심 팩트 | 출처 |
|---|---|---|
| 영업비밀 | 산업기술 유출범의 **88.3%가 내부자**, 유출 주체 **1위=퇴직자(51.2%)**. 5년 해외유출 추정 피해 **약 23조 원**. 퇴직자 비밀보호서약 체결 **44.8%뿐** | 경찰청·특허청·국정원 |
| 보안 | 퇴사자 계정 방치 → **역대 최대 6,246억 원 과징금(쿠팡, 2026.6)**. 퇴사 예고 후 **24시간 내 유출 720% 급증**. 퇴사 후 계정 **50%가 하루+ 잔존** | 개인정보위·Cyberhaven·OneLogin |
| 노무 | 임금체불 **2024년 2조 448억 원·28만 명**. 퇴직금 미정산은 **14일 경과 즉시 형사처벌 + 연 20% 지연이자** | 고용노동부 |

**처벌 공백 = 사전예방이 유일한 답**: 기술유출 기소율 **18.7%**, 피해 25조에 실형 누적 9건. → **"터지면 되돌릴 수 없다. 통제 가능한 유일한 지점이 퇴사 관문이다."**

**Why-now(법 개정 3연타)**: 영업비밀 침해 징벌 배상 **3배→5배**(2024.8, 세계 최고) · 상습 임금체불 근절법 **3배 배상·형사처벌**(2025.10) · 노란봉투법(2026.3). 세 법이 전부 "퇴사"라는 한 지점을 가리킨다. + 생성형 AI로 흩어진 비정형 기준을 자동 대조할 수 있게 된 첫 시점.

## 3. 어떻게 작동하는가

### 3-1. 프로세스 (퇴사 1건이 3레일을 관통)
```
[퇴사 이벤트 접수] → ① AI 케이스 인테이크(유형 분류: 자발/권고/해고/계약만료)
        │
   ┌────┼────┐
   ▼    ▼    ▼
 노무   영업비밀  보안
 법정기한  접근 핵심자산  계정·SaaS·자산
 절차 요건  특정→보호요건  회수 체크
 리스크알림  점검·재서약   (DLP 로그 연동)
   └────┼────┘
        ▼
 [통합 증적 패키지 자동 축적] → "이 퇴사는 방어 가능한 상태" 리포트 + PDF Export
```
레일별 게이트 판정 요소: 🟦 노무 = 금품청산 14일(§36)·서면통지(§27)·정산 증적 / 🟪 영업비밀 = 보호요건 3요소(비밀표시·재서약·접근제한) / 🟧 보안 = 계정·SaaS·기기 회수 + 오피스키퍼 로그 흡수.

### 3-2. 판단 기준의 원천 — 3층 기준 스택 (★해자의 정체)
**시스템이 기준을 만들지 않는다. 기준은 전부 공개 문서에서 오고, 회사 내부 사실과 대조될 뿐이다.**

| 층 | 원천 | 성격 |
|---|---|---|
| **L1 법령** | 국가법령정보센터(근기법·부정경쟁방지법 등) | 확정 숫자·형식 — 기계적 대조 (예: 금품청산 14일, 해고 서면통지) |
| **L2 판례·판정례** | 대법원 판례·중노위 판정례 | 해석 기준 — "요구 요소"를 추출해 체크항목으로 구조화 |
| **L3 정부 가이드** | 특허청·고용부 매뉴얼·표준서식 | 운영 체크리스트 (법적으로 가장 안전한 원천) |

- **AI(LLM)가 하는 것**: 비정형 문서에서 사실 추출(이 파일이 서약서인가/비밀표시 있는가) · 케이스↔기준 매핑 · 출처 링크 포함 근거 문장 생성.
- **AI가 하지 않는 것**: 기준의 창작, 종합적 법적 판단. (→ §4)

### 3-3. 템플릿 3층 모델 (제품의 척추 — 사업성의 근거)
- **L1 기본 라이브러리**(우리 제공, 판례·규정 큐레이션 = 해자) → **L2 회사 맞춤**(사용자가 항목 추가·편집 = 전환비용·고착) → **L3 케이스 즉석조정**(인테이크에서 AI 자동 + 수동).
- ★원칙: **"빈 종이 빌더" 금지 → Fork & Customize.** 사용자가 백지에서 만들게 하면 판례 큐레이션 해자가 붕괴한다. 항상 우리 기본 템플릿 위에 회사 항목을 얹는 방식.
- 레일별 템플릿 축이 다름: **노무=퇴사유형×직급 / 영업비밀=부서·직무 / 보안=부서×권한레벨.**

### 3-4. 오케스트레이터 모델 (주 사용자 = 인사총무팀)
- 인사담당자는 3레일을 혼자 실행 못 한다(영업비밀=개발팀 자산 모름, 보안=IT 권한 없음). → 인사가 **허브(오케스트레이터)**, 영업비밀·보안 레일은 담당팀에 **배분·추적**한다. 이 정보 비대칭이 곧 제품이 필요한 이유.

## 4. ⚖️ 절대 지켜야 할 법적 경계 (개발·문구·데모 전원 준수)

**공인노무사법 27조**가 금지하는 비노무사 업무 = ①대행·대리 ②노동관계법령 서류의 작성·확인 ③**노무관리진단**. → ExitGuard는 셋 다 하지 않는다. 위반 시 **3년 이하 징역 또는 500만 원 이하 벌금**(+ "오인될 표시·광고"까지 금지).

### 문구 규율 (제품 UI·발표·문서 전부)
| ❌ 절대 금지 | ✅ 허용 |
|---|---|
| "위법입니다" / "부당해고입니다" / "패소합니다" | "법정 기한 D-3" |
| "노무 진단" / "리스크 진단" / **'진단'이라는 단어 자체** | "리스크 알림" / "기한 관제" / "셀프 체크 지원" / "상태 대조" |
| "노무 상담" / "법률 자문" | "공개 판정례에서 이 요소 누락 시 부당해고로 판정된 **사례가 있습니다**" |
| AI가 일반인에게 생성형 법률 답변 | "전문가 확인을 권장합니다" / "기준 대비 미충족 항목 3건" |

- **추가 주의**: '진단' 용어 폐기만으론 부족. 노무관리진단의 법령 정의에 **"개선방안 제시"**가 포함되므로, "미충족 표시 + 보완항목 자동 생성"이 실질적으로 여기 근접하지 않게 설계할 것. LLM은 **공개 기준 인용·링크 위주**로 제한하고 생성형 판단은 배제.
- **판단이 필요한 분기마다 [전문가 연결] 버튼**을 세운다. 상용화 시 **노무법인 제휴(판단 주체=노무사, 우리=도구)**가 기본 구조.
- 근거: 로폼(자동작성 도구 = 적법) ✅ vs AI대륙아주(생성형 법률 답변 = 변협 징계) ❌ — 이 선을 넘지 않는다.

1. **세로 절단 1개.** 6뷰·전기능 나열 금지. **"핵심 개발자 김민수 퇴사 1건이 3레일을 관통 → 방어가능 리포트 + 증적 export"** 단일 시나리오를 진짜로 돌린다. (기능 나열이 예선 최대 사망 리스크였음)
2. **"실제 동작"의 경계선을 명확히**:
   - **진짜로 구현 (심사 신뢰도의 핵심)**: **AI 판례·판정례 대조 엔진**. 이게 돌아가야 "GPT 래퍼 아님 / 판단 안 함"이 증명된다. = 팀이 주장하는 유일한 기술 해자이자 STEP2의 핵심 목표.
   - **시뮬레이션 데이터로 고정**: DLP/오피스키퍼 로그, 계정·SaaS 회수 실행, 그룹웨어 자동연동. 실연동은 6주에 불가능하고 해자도 아니다. 여기 힘 빼지 말 것.
3. **데모 머니샷**: 부서·직급 다른 두 사람(예: 개발팀 팀장 vs 총무팀 사원)의 게이트가 극적으로 갈리는 대조 → "같은 퇴사인데 사람 따라 게이트가 딴판".

### 팀 & 역할 (대략)
- **조정호** — 백엔드 개발자 출신 / **심의지** —백엔드 개발자 출신 / **라하나** — QA출신
- ExitGuard 전용 AI 팀(에이전트, CEO=도비 등)은 이 레포 안에서 직접 구성.

### 심사 킬러질문 대비 (요지)
- *"체크리스트/엑셀이면 되잖아?"* → 엑셀이 못 하는 3가지: ①구두통보가 판정례상 지는 유형임을 아는 것 ②접근 자산 13건을 권한·로그에서 자동 특정 ③소송에서 쓸 봉인된 기록.
- *"보안회사가 왜 노무를?"* → 퇴사자 기술유출이야말로 보안의 정통 주제. 노무 레일은 영업비밀 증적을 완성하는 절반.
