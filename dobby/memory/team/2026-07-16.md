## 수정 — Phase 1 TS+SEC 명세 결함 2건 (도비 검증에서 발견) [작성: 헤르미온느]
도비가 착지된 6파일을 재검증하다 결함 2건을 짚어 헤르미온느에 수정 위임. 두 파일만 수정, 캘리브레이션 표·기타 내용은 불변.

- **① [중요] `docs/spec/영업비밀/data-model.md` §2-3 `protection_status` 파생식 표기 오류.**
  - 문제: `missing = { not secret_mark, not re_pledge, not access_control }` — `{}`를 Python `set()` literal로 구현하면 bool 값이 겹칠 때 **중복 제거**되어 `len()`이 실제 결여 개수를 반영 못함. 캘리브레이션 표(6행)와 공식이 자기모순. 예) 고객DB_스키마(✓✕✓, 실제 결여 1)를 set으로 계산하면 `{False,True,False}`→dedupe→`len=2`→`unmet` 오판정(표 정답은 `re_pledge_needed`). 사업계획_2026도 동일 오류. 론이 이 공식을 `compare.py`·TS 파생식 구현 시 캘리브레이션 테스트로 그대로 쓰기로 되어 있어 표기 그대로면 구현 버그로 착지할 위험.
  - 수정: `missing_count = int(not secret_mark)+int(not re_pledge)+int(not access_control)` 명시적 개수로 교체 + set 오독 주의 주석 추가. 분기 로직·순서는 동일 유지.
  - **6행 재검증(수정 공식 대입, 헤르미온느 재확인)**: 핵심도면_v7(✕✕✓→count2→`unmet`)·고객DB_스키마(✓✕✓→count1·re_pledge결여→`re_pledge_needed`)·API_인증키_모음(✓✓✕→access_control결여→`unmet`)·배포_스크립트_v3(✓✓✓→count0→`met`)·사업계획_2026(✕✓✓→count1·secret_mark결여→`mark_needed`)·인프라_구성도(✓✓✓→`met`). **6/6 전부 기존 표 정답과 일치.**
- **② [경미] `docs/spec/보안/api-spec.md` 응답에 `compare` 키 누락.**
  - 문제: 코어 §3 3레일 공통 뼈대 응답은 `{ rail, completion, items, compare, badges }`인데 SEC 응답 예시에 `compare`가 통째로 빠져 FE가 3레일을 동일 로직으로 순회 시 키 부재로 깨질 위험.
  - 수정: 상단 뼈대 설명·응답 예시에 `"compare": []` 추가 + "SEC는 대조(compare) 대상 아님 → 항상 빈 배열, 코어 공통 형태 유지용" 주석 명시.
- 상태: **두 파일 수정 완료. 커밋은 도비가 diff 직접 검증 후 진행(헤르미온느 커밋 안 함 — 도비 지시).** 스테이징 대상 = 이 두 spec 파일 + 본 팀 로그. `.claude/settings.json` 등 미관련 변경은 건드리지 않음.
