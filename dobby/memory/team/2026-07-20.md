### 데이터 성격·한계 (트레이드오프)

### ★TOP3 (도비 검증: 전부 타당 — 세로절단 종반부·해자 계약이 얇음)
- **T1 [보강] compare 엔진 LLM/결정론 경계 계약 부재** — 코어 data-model §6-3에 "LLM=인용·링크만" 제약은 있으나, **무엇을 LLM이(비정형 텍스트→신호추출·근거생성)/무엇이 결정론(요소→판례매핑·5행 shape)인지 역할분담 계약이 없음.** PRODUCT §3-2가 해자로 못박은 "AI 사실추출·매핑·근거생성"의 방어선이 구현자 재량으로 흔들림 = "GPT 래퍼 아님" 심사 1순위 질문 직결. → 코어 §6에 LLM 역할·입출력·직역법 제약 명시.
- **T2 [보강] 증적 자동 봉인 트리거 계약 부재** — api-spec에 수동 `POST /cases/{id}/evidence` 1건만(도비 확인). CM-12/PRODUCT §3-1의 "증적 **자동** 축적"이 미계약 — 각 처리(상신확인·회수확인·대조결과)가 언제 자동 Evidence append+prev_hash 체인 잇는지 규약 없음 + compare 결과 영속화·증적 편입 미정. → 코어 data-model §3-4/api-spec §2-5에 이벤트→자동봉인 규칙.
- **T3 [경량신설] 게이트 통과→퇴사 승인 확정 플로우 부재** — `Case.status`(in_progress/review_waiting/completed) 필드는 있으나 **전이 트리거·엔드포인트 없음**(도비 확인). `defensible=true`는 파생 boolean뿐. PRODUCT "세 레일 충족돼야 승인 강제"가 미구현. → Case 상태전이 규칙 + `POST /cases/{id}/approve`(경량).

### (B) 데모·심사 유리
- **B1 [보강] 방어리포트 뷰 스키마 부재** — CM-13 export가 "완료율·리스크·이력 포함" 한 줄뿐. DefenseReport 스키마(KPI·3레일 배지·판례인용·증적체인 요약·**boundary 고지**) 없음. 머니샷+외부제출 문서라 boundary 누락 시 직역법 위험.
- **B2 [보강] 프로파일·템플릿 시드 통합 계약** — 접수 시 items 자동생성 근원(김민준 profile→rail_map→3 RailTemplate→TemplateItem)이 레일별로 흩어짐. CM-22(빈종이 금지=해자)의 실체.
- **B3 [신설] CM-33 머니샷(두 케이스 게이트 대조)** — 두번째 시드 케이스+대조 명세 없음(엔드포인트는 `GET /cases` 재사용).

## ✅ 코어 계약 보강 T1·T2·T3·B1 + C3 완료 (헤르미온느 위임, 도비 diff 검증) [작성: 심의지(도비 위임)]

- **T2 증적 자동봉인(§3-4-1 + Evidence `event_type`·`origin` 필드 추가)** — 자동 트리거 6종(item_submitted/confirmed/rejected·compare_recorded·recovery_confirmed·case_approved)→서버가 mutation 동일 트랜잭션에서 auto append. 체인 freeze: `seq=직전+1`·`integrity_hash=SHA-256(canonical_json(payload)+prev_hash)`. compare 결과 스냅샷 봉인→리포트가 재계산·환각 없이 인용.
