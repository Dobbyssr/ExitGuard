심의지 대표님 지시로 "노무 데이터/API를 실제로 합법·기술적으로 가져올 수 있나"를 론에게 위임 조사(①) → 실물 확인(②)까지 완료. 도비 직접 검증 통과.

### 확정 소스 2종 (도비가 근거 URL·실물 직접 검증)
- **L1 법령 — 국가법령정보센터 OpenAPI (open.law.go.kr, 1순위/확실)**: "영리 목적 이용 포함 자유 활용 보장" 명시(도비 WebFetch 확인). 무료·출처표시 의무. 근기법 §36·§27·§26 조문 100% 커버. **인증키(OC) 발급 = 회원가입→활용신청→담당자 심의 1~2일**, 이건 대표님 개인계정 인증이라 도비 대행 불가. 단서: "개별 API별 제한 가능" — 판례(법원) target 상업이용만 개별 미확정(MVP엔 불필요).
- **L2 판정례 — 중노위 주요판정사례 (data.go.kr 15143186, 2순위)**: "이용허락범위 제한 없음 = 상업 이용 가능" 명시(도비 확인). 로그인 없이 CSV 즉시 다운로드 성공(오픈API XML/JSON도 실재하나 활용신청 승인 필요). 399건.

### L2 CSV 실측 (도비 PowerShell Import-Csv 재검증 — 론 4개 주장 전부 통과)
- 헤더: `구분·순번·자료구분·제목·위원회명·작성일자·조회수` / 399행 / 전부 "중앙"(지노위 없음).
- 자료구분 분포 검증 일치: 해고계열(징계87+통상43+경영상12)=**142건(36%)**, 부당노동행위 74 등.
- **§36 금품청산 = 0건 확정**(금품청산·퇴직금·14일·36조 매칭 전부 0) → 이 소스는 §36 미커버. **구조적 공백(§36=임금체불 형사·진정 영역, 중노위 부당해고 판정 대상 아님) → 문제 아님, §36은 법령 API로 커버.**
- **§27 서면통지 부당해고 사례 15건 실존.** 데모 직결 예시: 순번 388(2025, 서면 미교부→부당해고, 근기법 §27 직접 인용), 순번 51(2015, **휴대폰 문자 통보→서면통지 위반 부당** = 구두통보 시나리오에 최적).

### 데이터 성격·한계 (트레이드오프)
- **즉시 가능**: 출처 배지 + "공개 판정례에서 이 요소 누락 시 부당해고 판정 사례가 있습니다"(직역법 허용 프레이밍) 사례 인용 한 줄.
- **부족**: 사건번호 73% 결여(제목 텍스트 정규식 의존), 판정요지=한 줄 요약뿐(요구요소 세분 매핑엔 근거 얕음), 인용조문 구조화 필드 없음. → "체크항목별 요구요소 매핑"은 제목 텍스트 파싱 로직 별도 필요.
- 인코딩 함정(구현 시 인계): CSV 실제 바이트 = **CP949(EUC-KR)**(헤더는 UTF-8 표기, 오독 주의) + 제목에 콤마 포함 → **RFC4180 quote-aware 파서 필수**(naive split 금지).
