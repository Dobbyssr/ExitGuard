# dobby/memory — 작업 기록 (공동창업 3인 공유)

도비의 모든 의미 있는 작업·결정은 여기 남긴다. (SOUL 규칙) Claude Code 자체 기억 시스템은 쓰지 않는다 — 기록은 전부 이 레포 안에.

## 구조

```
team/YYYY-MM-DD.md          ← 팀 공유: 회사 차원 결정·마일스톤. 항목마다 [작성: 이름] 태그.
personal/조정호/YYYY-MM-DD.md   ← 개인 세션 로그 (사람별 폴더 = git 충돌 원천 차단)
personal/심의지/YYYY-MM-DD.md
personal/라하나/YYYY-MM-DD.md
```

- **개인 로그**: 접속한 대표님 본인 폴더에 그날 파일로. 그 세션의 작업 흐름·다음 액션.
- **팀 로그**: 다른 대표님도 알아야 할 결정만 `team/`에 append(`[작성: 이름]` 태그).

## 규칙

- 세션 시작 시 도비는 **team 최근 파일 + 접속자 개인 폴더 최근 파일**을 함께 읽어 맥락을 잇는다. (그 전에 `git pull`)
- "메모로 남길까요?"라고 묻지 않는다 — 그냥 남긴다.
- `dobby/` 변경은 도비가 자동 commit+push 해도 된다(팀 공유 목적). 상세 = `SOUL.md`.
