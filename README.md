# ExitGuard

> 퇴사하는 순간의 리스크를, 하나의 게이트로.
> **3-Rail Exit Gate** — 노무·영업비밀·보안 3중 리스크를 단일 콘솔에서 관제하는 인사·보안 담당자용 SaaS.

`지란지교 해커톤 2026 · STEP1` · 오피스키퍼(DLP)의 오프보딩 확장

문제 → 3-Rail 게이트 → 작동 방식 → 근거를 한 화면에 담은 프로젝트 랜딩 페이지.

- 퇴사는 하나의 이벤트가 아니다 — **노무·영업비밀·보안 3개 리스크가 동시에** 터진다
- 부서별 엑셀에 흩어진 마지막 체크를 **단일 관문·단일 책임자** 아래로 모은다
- **세 레일을 모두 통과해야 퇴사 승인** · 그룹웨어 read-only 연동 · 법정 근거 기반

## 문제 — 나가는 직원은 칼 두 개를 겨눈다
직원이 퇴사하는 순간, 회사는 두 가지 위협을 동시에 마주합니다.
- **유출** — 기술·영업비밀이 걸어 나간다 (퇴사 후 회사 계정 50% 잔존·OneLogin, 비밀보호서약 체결 44.8%)
- **소송** — 회사가 놓친 노동법을 근거로 되받아친다 (부당해고·임금·퇴직금)

문제는 이 리스크가 **노무·영업비밀·보안 3개 부서에 흩어져** 각자 다른 도구로 처리된다는 것.
누구도 "이 퇴사가 방어 가능한 상태인가?"를 한눈에 보지 못한다.

> 관리 체계를 다 갖춘 대기업도 뚫린다 — 2026.6 쿠팡, 퇴직자 계정을 통해 6,246억 원 규모 유출.

## 솔루션 — 3-Rail Exit Gate
퇴사 이벤트가 트리거되면 세 레일을 동시에 세우고, **셋 다 충족돼야 "방어 가능 상태(게이트 통과)"** 로 판정 → 증적 봉인·PDF Export.

| 레일 | 핵심 |
|---|---|
| 🟦 노무 | 금품청산 14일(§36)·서면통지(§27)·정산 증적 |
| 🟪 영업비밀 | 보호요건 3요소 — 비밀표시·재서약·접근제한 |
| 🟧 보안 | 계정·SaaS·기기 회수 + **오피스키퍼 로그 read-only 흡수** |

**AI 기준 대조 엔진**: 생성형 AI가 법령·판정례·가이드(비정형 기준)와 현재 상태를 자동 대조해 미충족 항목·법정기한(D-day)·필요 증적을 제시.

> 현재는 시나리오 기반 실동작 프로토타입이며, AI 엔진 본격 실장은 STEP2 목표입니다.

## 왜 지금 (Why now)
- 영업비밀 침해 징벌배상 3배→5배 (2024.8)
- 노란봉투법 시행 (2026.3)
- 국가핵심기술 해외유출 적발 1건(’21)→10건(’24)
- 생성형 AI로 흩어진 비정형 기준을 자동 대조할 수 있게 된 첫 시점
