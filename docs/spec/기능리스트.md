# ExitGuard 기능 리스트 (본선 MVP)

> **문서 성격** — ExitGuard 본선 MVP의 **기능 전량 도출 + 분류**. 리스트 레벨(기능ID·이름·1줄목적·액터·태그)까지만 다룬다. 각 기능의 상세 플로우·상태전이·수용기준은 이 리스트 확정 후 별도 Feature Spec으로 명세한다.
> **작성**: 헤르미온느(기획·명세) · 근거 SSOT: `dobby/PRODUCT.md`(§1·§3·§4·§6) + 원본 최종안(§3·§9). 신규 사실 창작 없음.
> **작성일**: 2026-07-14

## 0. 범례 (반드시 먼저 읽을 것)

### 기능 ID 접두어 (3레일 + 공통)
| 접두어 | 그룹 | 대응 레일 |
|---|---|---|
| `CM` | 공통(Common) | 인테이크·오케스트레이션·게이트·증적·템플릿·리포트·전문가연결·기준DB |
| `LB` | 노무(Labor) | 🧑‍⚖️ 노무 레일 |
| `TS` | 영업비밀(Trade Secret) | 🔐 영업비밀 레일 |
| `SEC` | 보안(Security) | 🖥️ 보안 레일 |

### 태그 2종 (전 기능 필수)
- **구현등급**: `[진짜구현]` = 본선에서 실제 동작 / `[시뮬데이터]` = 고정 시뮬레이션 데이터로 대체 (PRODUCT §6 경계 엄수).
  - 진짜구현 대상 = **AI 판례·판정례 대조 엔진, 3층 기준 DB, 결정론적 기한 계산, 게이트 집계, 증적 봉인·Export** (= 유일한 기술 해자).
  - 시뮬데이터 대상 = **DLP/오피스키퍼 로그, 계정·SaaS 회수 실행, 그룹웨어 자동연동, 권한/자산/계정 원천 데이터**.
- **MVP포함**: `[MVP]` = 세로절단 시나리오에 필요 / `[Post-MVP]` = 이번 6주 범위 밖.
- **★CP** = **Critical Path**. 세로절단 시나리오("김민수 퇴사 1건 → 3레일 관통 → 방어가능 리포트 + 증적 export")가 실제로 돌기 위한 **최소 필수 기능**. (§4 별도 정리)

### 근거 배지 (판정 요소가 있는 기능만)
- **L1 법령**(국가법령정보센터) / **L2 판례·판정례**(대법원·중노위) / **L3 정부가이드**(특허청·고용부 매뉴얼·표준서식). PRODUCT §3-2 3층 기준 스택.

### 액터
- **인사총무**(오케스트레이터/허브) · **담당자**(영업비밀·보안 레일 수행) · **승인자**(게이트 진행 승인). PRODUCT §3-4 오케스트레이터 모델.
- **인증 역할은 admin/user 2역할**로 단순화(CM-17). 관리자(admin)가 일반사용자(user)에게 승인·레일 수행 권한을 부여하는 **권한 부여 기반 흐름**. 위 업무 액터(인사총무·담당자·승인자)는 이 2역할에 권한으로 매핑된다.
- **⚙️ 확장성 주석**: 5~100인 중소기업은 인사담당자 1명이 3레일을 관리하는 경우가 많아 **영업비밀·보안 레일 액터를 단일 "담당자"로 통합**했다. 단 **데이터모델·역할 배분 구조는 "영업비밀 담당팀 / 보안팀" 3팀 분리로 확장 가능하도록 설계 여지를 남긴다**(대기업/역할 분리 조직 대응). 즉 지금은 1인 담당자로 표기하되, 레일별 담당 역할 필드는 별도로 두어 후일 분리·배분할 수 있게 한다.

### ⚖️ 문구 규율 (이 문서 전체 적용 — PRODUCT §4)
금지어("진단"·"위법입니다"·"부당해고입니다"·"패소합니다"·"노무상담/법률자문"·AI 생성형 법률판단) 미사용. 판정 관련 표현은 전부 **"상태 대조 / 기한 관제 / 리스크 알림 / 기준 대비 미충족 N건 / 공개 판정례에 …사례가 있습니다"** 프레이밍으로만 기술.

## 5. ★ Critical Path — 세로절단 최소 기능 집합

**시나리오(단일·고정)**: "핵심 개발자 **김민수** 퇴사 1건 접수 → 3레일 관통 → 미충족 보완 → **'방어 가능 상태' 리포트 + 증적 Export**".
아래 기능이 **모두** 있어야 이 시나리오가 처음부터 끝까지 돌아간다. 하나라도 빠지면 세로절단 실패.

```
[접수]  CM-01 인테이크 → CM-02 유형분류
          │  (기준 근거: CM-08 3층 기준 DB · CM-09 근거 배지 · CM-10 기본 템플릿 · CM-15 시뮬 시드)
   ┌──────┼──────────────────────┐
   ▼      ▼                       ▼
 노무    영업비밀                  보안
 LB-01 타임라인   TS-01 자산 특정        SEC-01 회수 체크리스트
 LB-02 D-day 관제 TS-02 보호요건 3요소    SEC-02 이상 반출 이력
 LB-03 절차 대조  TS-05 비밀관리성 대조
 LB-04 판정례 대조
   └──────┼──────────────────────┘
          ▼
 CM-05 대시보드 → CM-04 게이트 판정 → CM-06 증적 봉인 → CM-07 리포트 Export
```

**Critical Path 최소 기능 (18개)**:
`CM-01` `CM-02` `CM-04` `CM-05` `CM-06` `CM-07` `CM-08` `CM-09` `CM-10` `CM-15` · `LB-01` `LB-02` `LB-03` `LB-04` · `TS-01` `TS-02` `TS-05` · `SEC-01` `SEC-02`

**데모 머니샷 추가 필요분**: `CM-14`(두 케이스 게이트 대조) + `CM-11a`(템플릿 읽기전용 노출, 해자 시연) — "같은 퇴사인데 사람 따라 게이트가 딴판"을 보이려면, 부서·직급 다른 2건이 시드에 있어야 하므로 `CM-15` 시드에 개발팀 팀장(김민수) + 총무팀 사원 2건을 포함해야 한다.

**CP 미포함 판단(MVP지만 CP 밖)**:
- `CM-17`(인증·권한 부여) — MVP로 확정됐으나 **데모는 인증된 상태로 시작**할 수 있어 세로절단 pierce-flow 구동에 load-bearing 아님. 따라서 CP에서 제외. (도비 판단으로 "데모를 로그인 화면부터 시연"으로 정하면 CP로 승격 — §7 참조)
- `CM-11a`(템플릿 읽기전용 노출) — 해자 시연용 MVP지만, 시나리오 대조·게이트는 `CM-10` 기본 템플릿으로 돈다. 데모 보강 기능으로 분류.

**Critical Path 밖(있으면 좋음, 이번 스코프 흔들면 후순위)**: CM-03(배분 추적) CM-12(즉석 조정) CM-13(전문가 연결 stub) CM-16(알림 센터) CM-17(인증) LB-05 LB-06 TS-03 TS-04.

## 6. 구현등급·MVP 집계

**총 36개** (공통 CM 19 · 노무 LB 7 · 영업비밀 TS 6 · 보안 SEC 4)

- **진짜구현 (28개)**: CM-01~10·11a·11b·12·14·16·17·18, LB-01~07, TS-02·03·05·06 → 판례·판정례 대조 엔진·3층 기준 DB·결정론적 기한 계산·게이트 집계·증적 봉인이 전부 여기(= 기술 해자).
- **분리 표기 (1개)**: **TS-01 접근 자산 자동 특정** = `[로직=진짜구현 / 입력=시뮬데이터]`. 권한·로그 → 자산목록 도출 *로직*은 진짜 구현(심사에서 "자산 특정도 엔진"으로 방어), *입력 데이터*는 시뮬.
- **시뮬데이터 (7개)**: CM-13·15, TS-04, SEC-01~04 → DLP/오피스키퍼 로그·계정회수 실행·자산/권한 원천·그룹웨어 연동·전문가 전달 채널. **여기 개발 리소스 최소화**(PRODUCT §6-2).
- **MVP (30개)**: CM-01~10·11a·12~17, LB-01~06, TS-01~05, SEC-01·02.
- **Post-MVP (6개)**: CM-11b·18, LB-07, TS-06, SEC-03·04.

## 7. 결정 이력 & 남은 `[결정필요]`

### ✅ 확정 완료 (대표님 결재 2026-07-14)
1. **CM-17 사용자 인증 → MVP 승격 + 역할 2개 단순화.** admin/user 2역할. 관리자가 일반사용자에게 승인·레일 수행 권한을 부여하는 권한 기반 흐름. 업무 액터(인사총무·담당자·승인자)는 이 2역할에 매핑. (CP는 미포함 — §5 판단)
2. **영업비밀·보안 레일 액터 → 단일 "담당자"로 통합.** 5~100인 중소기업은 인사담당자 1명이 3레일 관리하는 경우 다수. 단 **데이터모델·역할 배분 구조는 3팀(영업비밀 담당팀/보안팀) 분리로 확장 가능하게 설계 여지 유지**(§0 범례 확장성 주석).
3. **CM-11 → 분할.** CM-11a 읽기전용 노출 = MVP(해자 시연), CM-11b 편집 UI = Post-MVP.
4. **CM-13 전문가 연결 → 버튼+케이스요약 핸드오프 stub까지 MVP** (실제 전달 채널=Post-MVP). 목적문에 "stub" 명시.
5. **TS-01 자산 특정 → 분리 표기** `[로직=진짜구현 / 입력=시뮬데이터]`. 심사 방어용.
