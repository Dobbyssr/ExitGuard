## SEC-01 ★CP — 계정·SaaS·기기 회수 체크리스트 (상신 → 확인)

**목적**: 퇴사자 계정·외부 SaaS·기기 회수 항목을 체크리스트로 제시하고 미회수를 표시하며, 대상자 회수 상신 → 관리자 확인 흐름으로 회수 사실을 기록한다.

**액터**: 담당자(대리 상신) · 관리자(확인)

**사전조건**: 케이스 접수(CM-04)·수집(CM-06) 완료. 회수 항목이 템플릿에서 `Item`(rail=security)으로 생성됨.

**트리거**: 레일상세(CM-07) 보안 탭 진입.

**기본 플로우**:
1. 회수 항목 6건 리스트 렌더(data-model §2): 이메일(C-11)·VPN(C-12)·GitHub(C-01)·Slack(C-21)·Figma·Notion(C-02)·노트북·보안키(C-31). 각 항목 `recovery_category`(계정/SaaS/기기).
2. 항목 상태 아이콘: 완료(`approved`)·상신검토필요(`submitted`)·대기(`pending`).
3. 미회수(`submitted`/`pending`) 항목은 [상신 검토] 진입 → 코어 `POST /items/{id}/submit`(대상자 회수 상신) → `POST /items/{id}/review`(관리자 확인).
4. 회수 진행률 표시(참고, SEC-03): 계정 2/3 · SaaS 1/2 · 기기 2/2 → **88%**.

**입력**: 회수 항목 상태(시뮬 원천). **출력**: `items` + `recovery_progress`(`GET /cases/{id}/rails/security`).

**데이터**: 코어 `Item`(rail=security) + `detail`(data-model §2), 코어 `Approval`(상신-검토).

**상태전이**: 코어 §4 그대로 — `pending→submitted`(회수 상신) → `approved`(관리자 확인 완료) / `rejected`(반려→재상신). GitHub(C-01)·Figma·Notion(C-02)가 데모의 `submitted→approved` 실연출 대상.

**수용 기준(AC)**:
- AC-1: 회수 항목 6건이 데모 code/name/category verbatim으로 존재.
- AC-2: 대상자 회수 상신 → 관리자 확인 시 항목 `submitted→approved` 전이(코어 상태머신).
- AC-3: 응답 `rail_notice == "체크리스트는 회수 실행이 아닌 확인 기록입니다."` (레일 상세 배너 — `CompareResult.boundary_notice`와 구분, 코어 §9-P C3).
- AC-4: `recovery_progress`가 계정 2/3·SaaS 1/2·기기 2/2·88%.

**문구경계**: 
- ❌ 금지: "계정을 회수(폐쇄)했습니다"(실행 단정) · "진단".
- ✅ 허용: "회수 확인/반납 확인" · "미회수 표시" · "확인 기록". 실제 실행은 담당팀·연동(시뮬)의 몫임을 명시.

## SEC-02 ★CP — 이상 반출 이력 표시

**목적**: 퇴사 통보 후 최근 30일 내 대용량 다운로드 등 이상 반출을 오피스키퍼/DLP 로그(시뮬)에서 표시해 영업비밀 유출 리스크 신호를 제공한다.

**액터**: 인사총무

**사전조건**: 케이스 접수·수집 완료.

**트리거**: 보안 탭 진입.

**기본 플로우**:
1. 이상반출 카드 렌더(data-model §3, 시뮬): "퇴사 통보 후 **최근 30일** 내 대용량 다운로드 **1건 (4.2GB)** 이 감지되었습니다."(데모 verbatim).
2. [로그 상세] 버튼(시뮬 stub).

**입력**: DLP 로그(시뮬 고정 시드). **출력**: `anomaly_exports[]`(`GET /cases/{id}/rails/security`).

**데이터**: `AnomalyExportLog`(data-model §3). `is_sim=true`.

**상태전이**: 없음(조회 전용).

**수용 기준(AC)**:
- AC-1: `anomaly_exports`에 4.2GB·30일·1건 데모값 정확히 표시.
- AC-2: `is_sim=true`(DLP 실연동 아님).

**문구경계**: "이상 반출 **이력/감지**" = 로그 표시. "유출 범죄다/위법이다" 단정 금지. 리스크 신호 알림 프레이밍만.
