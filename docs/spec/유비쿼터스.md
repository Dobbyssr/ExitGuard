## 11. 유비쿼터스 언어 정합 규칙

- SQLModel 클래스명 = 엔티티 코드(`Case`,`Item`…). 필드명 = data-model 필드명 그대로(snake_case).
- FastAPI 경로/라우터 = api-spec 경로 그대로. Pydantic 스키마 = data-model shape.
- enum 값은 위 표 코드 문자열 그대로(대소문자·언더스코어 포함).
- 레일 서비스 분할: `case` / `rails`(labor·trade_secret·security) / `compare` / `evidence` (PRODUCT 도메인 향 3단).

## 13. ⚖️ 직역법 용어 규율 (전 문서·코드·UI 강제)

- **금지어**: "진단"(단어 자체) · "위법입니다" · "부당해고입니다" · "패소합니다" · "노무 상담/법률 자문" · AI 생성형 법률 판단.
- **허용 프레이밍**: "미충족 N건" · "기한 D-day / D-3" · "리스크 알림" · "상태 대조" · "…로 판정된 **사례가 있습니다**" · "전문가 확인을 권장합니다".
- `CompareResult.boundary_notice` **고정 문구**(data-model §6-3): *"본 결과는 위법 여부를 판정하지 않습니다. 공개된 법령·판례 기준과 회사 입력 상태의 대조 결과·기한만 제공하며, 구체 사건의 판단은 전문가의 몫입니다."*
- `risk` 행은 단정 금지(사례 프레이밍). `status` 행은 "개선방안 제시" 어조 금지(노무관리진단 정의 회피).

## 14. 레일 확장 예약 지점 (대표2·3 참조)

대표2(노무)·대표3(TS+SEC)가 이 척추만 보고 착수할 수 있는 예약 지점:
1. **레일 enum**: `labor`/`trade_secret`/`security` 고정(§2).
2. **항목 상세 슬롯**: `Item.detail` / `TemplateItem.detail_schema`(json) — 레일 고유 필드.
3. **compare 규칙**: shape는 §6 고정, 레일별 신호→기준 매핑만 `compare.py`에 구현.
4. **근거 시드**: `Standard`에 레일별 L1/L2/L3 레코드 추가.
5. **레일 엔드포인트**: `GET /cases/{id}/rails/{rail}` 뼈대 예약(api-spec §3), 공통 뼈대 응답 형태 고정.
6. **공통 재사용**: 상신-검토·게이트·증적·배지·상태머신은 신설 금지, 코어 재사용.
